# cc-statusbar 0.4.0 — Context-window usage % (spec)

Status: **approved, not yet implemented** (owner decision 2026-05-31). Build in a
fresh session.

## Goal

Show what share of the **model's context window** is currently in use in the
active (Lead) session — so the user knows how large a next task can be.

This is **different** from the existing cost metric (cumulative tokens spent,
which only grows). Context usage reflects how full the model's current prompt
window is right now.

## Source of the window limit — Models API ONLY (no fallback table)

**Owner decision (2026-05-31): NO hardcoded model→window table.** A stale table
would silently feed wrong limits after models change and we forget it exists.
"Fail visibly, fix by update" beats "silently wrong". Same philosophy as the
quota reliability caveat already in the README.

- Read the model id from the transcript: `message.model` (e.g. `claude-opus-4-8`).
- `GET https://api.anthropic.com/v1/models/{id}` using the **same local OAuth
  token** as the quota feature (`~/.claude/.credentials.json`,
  `anthropic-beta: oauth-2025-04-20`, `anthropic-version`).
- Use the response field **`max_input_tokens`** as the window limit.
  (Confirmed in Anthropic docs: the Models API returns `max_input_tokens`,
  `max_tokens`, `capabilities` per model.)
- **Cache** the result per model id for a long time (limits don't change) — e.g.
  24h, persisted in `globalState`. One cheap call, throttled.
- If the call fails / model not returned / auth changed → **do NOT guess**. Hide
  the context line and show a short note ("context limit unavailable"). Fixed
  via a plugin update.

### ⚠️ Verify first at implementation

- Confirm the **OAuth (subscription) token can call `GET /v1/models/{id}`**. If
  the OAuth scope only allows `/v1/messages` (as used for quota headers), this
  approach needs an alternative. If `/v1/models` is not accessible with the
  OAuth token → that is exactly the "fail visibly" path (hide the line) until a
  fix. Investigate before committing to the design.
- Confirm `max_input_tokens` value matches the window Claude Code actually uses
  (Opus 4.8 → 1,000,000; cross-check against observed context, which already hit
  ~468k in a real Opus 4.8 session → consistent with 1M).

### Reference (authoritative, fetched 2026-05-31)

- Opus 4.8 → **1M**, Sonnet 4.6 → **1M** (beta-labelled in the news post),
  Haiku 4.5 → **200k**. Source: Anthropic Models overview. These are for
  cross-checking only — the runtime value comes from the Models API.

## Current context size (numerator)

- = the **last assistant message of the MAIN transcript**:
  `input_tokens + cache_read_input_tokens + cache_creation_input_tokens`.
- **MAIN transcript only.** Subagents (`subagents/agent-*.jsonl`) have their OWN
  separate context windows — their internal work does NOT consume the Lead's
  window. Exclude them here. (Note: the *cost* metric sums subagents; the
  *context %* must NOT — two different computations.)
- This is the **real** number the model received, read from the transcript — not
  an estimate. So thinking/effort levels are automatically accounted for (they
  change output/cost, not the recorded input size; the window limit itself is
  fixed per model and unaffected by thinking/effort).

## Display

- Tooltip + panel line: `context: X% (used / limit)` — e.g.
  RU `контекст: 47% (468k / 1M)`, EN `context: 47% (468k / 1M)`.
- Colour when high (e.g. ≥85% → warning colour) — optional.
- Collapsed status-bar stays tariff-only; context % lives in tooltip + panel.
- When limit unavailable: show `context: 468k (limit n/a)` — used tokens still
  shown (we know them), but NO misleading %.

## i18n keys (new)

- `contextLine(used, limit, pct)`, `contextNoLimit(used)`,
  `contextLimitUnavailable` — EN + RU.

## Reliability (README)

Add a short note mirroring the quota caveat: the context % depends on the Models
API + local OAuth token; if that channel changes, the context line hides until a
plugin update — local cost/cache metrics are unaffected.

## Tests

- limit-unavailable → used shown, no `%`.
